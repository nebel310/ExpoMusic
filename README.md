# ExpoMusic Backend

Музыкальный сервис на FastAPI.  
Первый pet-проект, в котором отрабатывались основы FastAPI, асинхронная работа с PostgreSQL и проектирование REST API. Бэкенд полностью функционален, фронтенд находится в стадии экспериментов (доступны динамические HTML/JS-страницы) и никогда из неё не выйдет, ведь я бекендер :)

## Основные возможности

**Пользователи и авторизация**
- Регистрация с подтверждением почты (через SMTP).
- Вход по JWT (access + refresh токены).
- Выход с аннулированием токенов (blacklist).
- Текущий профиль пользователя.
- Тестовый администратор создаётся автоматически при первом запуске.

**Треки и жанры**
- Просмотр всех треков и получение трека по ID.
- Загрузка новых треков (только авторизованные).
- Получение треков по жанру.
- Управление жанрами (создание, просмотр).
- Удаление треков (заглушка – только для администраторов).

**Плейлисты**
- Создание, редактирование и удаление плейлистов (только владелец).
- Добавление и удаление треков из плейлиста.
- Получение плейлиста с полным списком ID треков.
- Публичные и приватные плейлисты.

**Медиатека**
- Избранное: добавление, удаление, просмотр любимых треков.
- «Не нравится»: отдельный список треков с возможностью удаления.
- Сохранённые плейлисты (saved playlists) – добавление, удаление, просмотр.

**Поиск**
- Глобальный поиск по трекам, плейлистам и жанрам.
- Настраиваемые лимиты выдачи.

**Документация OpenAPI**
- Полное описание эндпоинтов доступно после запуска по адресу `/docs` (Swagger UI).

## Схема базы данных

### users
| Поле            | Тип        | Описание                       |
|-----------------|------------|--------------------------------|
| id              | int (PK)   | Уникальный идентификатор       |
| username        | varchar    | Имя пользователя               |
| email           | varchar    | Уникальный email                |
| hashed_password | varchar    | Хэш пароля                     |
| is_confirmed    | boolean    | Флаг подтверждения почты       |
| is_admin        | boolean    | Роль администратора            |
| created_at      | timestamptz| Дата регистрации               |

### refresh_tokens
| Поле       | Тип         | Описание                         |
|------------|-------------|----------------------------------|
| id         | int (PK)    | Идентификатор токена             |
| user_id    | int (FK)    | Внешний ключ к users.id          |
| token      | varchar     | Уникальный refresh-токен        |
| expires_at | timestamptz | Срок действия токена            |
| created_at | timestamptz | Дата создания                    |

### blacklisted_tokens
| Поле       | Тип         | Описание                         |
|------------|-------------|----------------------------------|
| id         | int (PK)    | Идентификатор записи             |
| token      | varchar     | Заблокированный токен           |
| expires_at | timestamptz | Срок истечения токена           |
| created_at | timestamptz | Дата добавления в чёрный список |

### tracks
| Поле         | Тип        | Описание                         |
|--------------|------------|----------------------------------|
| id           | int (PK)   | Уникальный идентификатор трека   |
| uploaded_by  | int (FK)   | Внешний ключ к users.id          |
| title        | varchar    | Название трека                   |
| artist       | varchar    | Исполнитель                      |
| genre_id     | int (FK)   | Внешний ключ к genres.id         |
| uploaded_at  | timestamptz| Дата загрузки                    |

### genres
| Поле | Тип        | Описание               |
|------|------------|------------------------|
| id   | int (PK)   | Идентификатор жанра    |
| name | varchar    | Название жанра (уник.) |

### playlists
| Поле       | Тип        | Описание                      |
|------------|------------|-------------------------------|
| id         | int (PK)   | Идентификатор плейлиста       |
| user_id    | int (FK)   | Внешний ключ к users.id       |
| name       | varchar    | Название плейлиста            |
| is_public  | boolean    | Флаг публичности              |
| created_at | timestamptz| Дата создания                 |

### playlist_tracks
| Поле        | Тип     | Описание                                |
|-------------|---------|-----------------------------------------|
| playlist_id | int (PK, FK) | Внешний ключ к playlists.id (составной PK) |
| track_id    | int (PK, FK) | Внешний ключ к tracks.id                |

### favorite_tracks
| Поле       | Тип        | Описание                         |
|------------|------------|----------------------------------|
| id         | int (PK)   | Идентификатор записи             |
| user_id    | int (FK)   | Внешний ключ к users.id          |
| track_id   | int (FK)   | Внешний ключ к tracks.id         |
| created_at | timestamptz| Дата добавления                  |

### disliked_tracks
| Поле       | Тип        | Описание                         |
|------------|------------|----------------------------------|
| id         | int (PK)   | Идентификатор записи             |
| user_id    | int (FK)   | Внешний ключ к users.id          |
| track_id   | int (FK)   | Внешний ключ к tracks.id         |
| created_at | timestamptz| Дата добавления                  |

### saved_playlists
| Поле        | Тип        | Описание                         |
|-------------|------------|----------------------------------|
| id          | int (PK)   | Идентификатор записи             |
| user_id     | int (FK)   | Внешний ключ к users.id          |
| playlist_id | int (FK)   | Внешний ключ к playlists.id      |
| created_at  | timestamptz| Дата сохранения                  |

Все связи обеспечиваются внешними ключами. При удалении трека или плейлиста соответствующие записи в связующих таблицах, а также в избранном и «дислайках» удаляются каскадно.

## Быстрый запуск (Docker Compose)

1. Убедитесь, что установлены Docker и Docker Compose.
2. Склонируйте репозиторий:
   ```bash
   git clone <url-репозитория>
   cd ExpoMusic
   ```
3. (Опционально) Проверьте или измените переменные окружения в `backend/.env`.  
   Файл уже содержит рабочую конфигурацию, включая тестовые учётные данные SMTP.
4. Запустите проект:
   ```bash
   docker compose up -d --build
   ```
5. После запуска:
   - Backend API доступен на `http://localhost:3001`
   - Swagger UI с полным описанием методов: `http://localhost:3001/docs`
   - База данных создаётся и наполняется тестовым контентом автоматически.

**Тестовый пользователь (администратор)**  
Email: `user@example.com`  
Пароль: `string`

Фронтенд в этом запуске не обслуживается. Вы можете вручную открыть `frontend/index.html` в браузере. Если вдруг это читает любопытный фронтендер, которому скучно, то я буду рад видеть фронт к моему клону спотифай :-)

## Стек технологий (backend)

- FastAPI (Python)
- SQLAlchemy (asyncpg) + PostgreSQL 17
- JWT-авторизация (python-jose, passlib)
- Подтверждение почты через SMTP (itsdangerous, smtplib)
- Docker Compose для развёртывания
